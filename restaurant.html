Restaurant Ordering Website

A full-stack ordering platform for a fictional restaurant — a live menu, real-time availability, and order processing, all driven by a REST API instead of hardcoded front-end data.

GitHub — code private No public deployment
Problem

Menus shouldn't need a redeploy to change.

A common shortcut in student and prototype projects is to hardcode the menu directly into the front-end — fast to build, but it means every price change, sold-out item, or new dish requires editing and shipping code. This project replaces that with a proper data layer: menu items live in a database and are fetched dynamically, so the storefront reflects reality without a redeploy.

Architecture

How it's put together

A React front-end renders the menu and order flow from live API data. An Express.js REST API sits in front of MongoDB, with Mongoose managing schemas for menu items and orders.

Technical Decisions

Why it's built this way

Data layer

  • MongoDB for a flexible document store that fits menu items with varying fields (add-ons, sizes, tags)
  • Mongoose schemas to keep validation and structure consistent across menu and order documents
  • REST endpoints separate menu reads from order writes, so the storefront stays fast under load

App layer

  • React with Hooks and component state for the menu grid, cart, and order flow
  • Every screen fetches from the API on load — no hardcoded arrays anywhere in the front-end
  • Responsive layout with HTML/CSS/JSX so the same views work on mobile ordering
Challenges & Trade-offs

What was hard

Moving off hardcoded data meant every component had to handle loading and empty states gracefully — a menu item can now be sold out, missing an image, or mid-update, and the UI needed to degrade sensibly instead of assuming the data was always complete.

The trade-off was added complexity in the front-end: components that used to just render a static array now need fetch logic, loading states, and error handling. That cost paid off in flexibility — the menu can change without touching the deployed front-end code.
